# 传输层

将连接、监听、请求逻辑封装为统一抽象。对于 Aha 来说主要用于适配器，当然也可以独立使用。

## 已有传输层

### core.transports.WebSocketClient

基于 [websockets](https://github.com/python-websockets/websockets) 的 WebSocket 客户端。

#### `open` 异步方法参数

| 参数名 | 类型 | 描述 |
| --- | --- | --- |
| uri | str |  |
| extra_headers | dict 可选 | 可视为 `additional_headers`。 |
| retry_config | dict 可选 | 用于 [`tenacity`](https://github.com/jd/tenacity)。需要为通过 `BaseBot.parse_retry_config` 类方法处理的配置，其会将类似 `{"stop_max_attempt_number": {"max_attempt_number": 3}}` 转换成 `{"stop": stop_max_attempt_number(max_attempt_number=3)}`。 |
| **kwargs | | 不与上述参数冲突的[其余参数](https://websockets.readthedocs.io/en/stable/reference/asyncio/client.html#websockets.asyncio.client.connect)、 |

**`listen` 返回**：bytes

**`invoke` 唯一参数**：Buffer | AsyncIterable[Buffer]

**`invoke` 始终返回 None**

## 使用

创建实例时接受 `disconnect_cb` 和 `reconnect_cb` 两个异步回调函数，分别在连接断开和重新连接时调用。

```python
import asyncio
from core.transports import WebSocketClient

client = WebSocketClient()
await client.open("wss://echo.websocket.org")  # 连接
listener = asyncio.create_task(client.listen(on_message))  # 启动监听任务
await client.invoke(b"Hello, WebSocket!")  # 发送一条消息
await asyncio.sleep(2)
await client.close()  # 关闭连接
await listener  # 等待监听任务结束
```

## 自定义传输层

客户端需满足 `core.transports.ClientTransport` 抽象，否则最起码应满足 `core.transports.Transport` 抽象。暂未实现与 FastAPI 服务对接的传输层抽象，预期为 `core.transports.FastAPITransport`。
