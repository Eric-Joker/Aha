# FAQ

## 模块

### 如何调试模块？

1. **日志**：使用 `logging.getLogger()` 获取 logger，Aha 会将日志输出到控制台和文件（`logs/` 目录），且添加了比 `INFO` 略详细但没 `DEBUG` 详细的 `AHA_DEBUG` 日志等级。
2. **事件匹配调试**：可在表达式中使用 `.apply(print)` 等技巧查看中间值，或使用[表达式的调试功能](./事件匹配表达式.md#调试)。

### 我的模块没有被 Aha 加载？

确认以下几点：

1. 模块目录或文件位于 `modules/` 下，若为目录需包含 `__init__.py` 文件。
2. 模块目录或文件名不能以 “DISABLED” 开头。
3. 如果使用了 `--load-only` 启动参数，确认模块名在列表中。
4. 检查 Aha 启动日志中是否有该模块相关的错误信息。

### 模块热重载后行为异常？

Aha 的模块重载机制通过重新导入 Python 模块实现。注意：

- 会重载所有 Aha 模块。
- 模块级全局变量会被重新初始化。
- 装饰器注册的回调会重新注册（旧的回调注册会先被清空）。
- 数据库模型定义建议放在模块目录的 `database.py` 或 `database/` 子目录下 —— 这两个位置在重载时**不会**被重新导入，避免 SQLAlchemy 的映射冲突。
- 如果使用[计划任务](./内置轮子与最佳实践/计划任务.md)的持久调度器，注意应固定 id 以每次都覆写任务。
- 热重载机制不建议用于生产环境。

### 如何让回调仅在特定平台运行？

在回调中使用[事件匹配表达式](./事件匹配表达式.md)的 `Pplatform` 字段：

```python
from core.expr import Pmsg, Pplatform

@on_message(Pmsg == "ping", Pplatform == "QQ")
async def _(event: Message):
    # 仅在 QQ 平台响应
    ...
```

或在回调内部通过 `event.platform` 判断。

## 事件与回调

### 回调没有被触发？

按以下顺序排查：

1. **匹配条件是否正确？** 检查字符串匹配是否大小写正确（比如 `on_message(Pmsg == "hello")` 是区分大小写的）。
2. **消息是否带前缀？** 如果使用了 `Pprefix == True`，确认消息包含[配置中的前缀](./内置轮子与最佳实践/消息前缀.md)或 @机器人。
3. **是否被黑白名单过滤？** 检查[配置文件中的群组/用户黑白名单](../安装与使用.md#面向平台用户群组的黑白名单)。
4. **是否被限速？** 如果启用了全局限速（配置文件中 `aha.limit`），超频的消息会被 `PM.limit` [默认表达式](./事件匹配表达式.md#默认表达式)过滤。
5. **回调函数是否为异步？** 所有回调必须是 `async def`。
6. **进行调试** [参见](#如何调试模块？)

### 同一个消息触发了多个回调？

这是正常的。Aha 的事件分发器会将事件分发给**所有**匹配的回调，而不是只分发给第一个匹配的。

如果你希望实现"回调优先级"，请自行利用同步源语实现。

## 数据与存储

### SimpleStore 和 SQLAlchemy 该用哪个？

只推荐 [SQLAlchemy 方案](./内置轮子与最佳实践/长效数据.md#通过-sqlalchemy-操作数据库)，[SimpleStore 方案](./内置轮子与最佳实践/长效数据.md#simplestore不推荐) 只适用于觉得数据库麻烦的时候。

### 如何在不同模块间共享数据？

有以下几种方式：

1. **直接 import**：模块 A 导出变量/函数，模块 B 导入使用。简单直接。
2. **External 事件**：通过[发布事件](./订阅与发布事件.md#发布事件)在模块间传递数据，解耦模块依赖。
3. **数据库**：要共享持久化数据，建议 import 另一个模块的映射对象。当然直接用表名也不是不行，记得 Aha 会自动添加模块名前缀就好。

### 数据库表没有被自动创建或修改？

重启 Aha。详见[长效数据](./内置轮子与最佳实践/长效数据.md#数据结构更新)。

## API 调用

### `NotImplementedError`

这说明当前使用的协议适配器没有实现该 API。不同适配器支持的 API 不同。请查阅[已有适配器](../已有适配器/README.md)文档或适配器源码确认可用 API。

### 如何在事件上下文之外调用 API？

见[向协议服务请求](./向协议服务请求/README.md#指定-bot-实例)。

### `APIException`

`APIException` 表示协议平台返回了错误或适配器认为返回的内容非法。检查：

- `exception.code`：错误码
- `exception.err_msg`：错误信息
- 确认账号权限是否满足操作要求（如是否管理员、是否群主等）

## 本地化

### 如何支持用户切换语言？

通过协议适配器配置中的 `lang` 键指定其默认语言。

模块本身无需处理语言切换逻辑 —— 只需在回调中声明 `localizer` 关键字参数即可自动获取触发回调所用语言的翻译器。

## 线程与异步

### 回调函数必须是异步的吗？

所有通过 `on_message`、`on_notice`、`on_request`、`on_meta`、`on_external` 注册的回调必须是 `async def` 异步函数。

通过 `on_start`、`on_cleanup` 注册的回调既可以是同步函数也可以是异步协程。

### free-threading

当配置文件 `aha.execution_mode` 设为 `thread` 时，部分回调可能在子线程中执行。如果回调逻辑不保证线程安全，在装饰器中声明 `threadable=False`：

```python
@on_message("test", threadable=False)
async def _(event: Message):
    # 只会在主线程执行
    ...
```
