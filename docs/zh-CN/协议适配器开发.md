# 协议适配器开发

本系列文档主要提供流程示例且减少描述，要求读者应：
- 具备将一个东西转换成另一个东西的逻辑设计能力；
- 具有 Aha 源码阅读（无需理解）能力；
- 通读[开发模块](./模块开发/README.md)涉及的所有文档；
- 可熟练使用 Python；
- 知晓 Python 的设计哲学，无需知晓 Aha 的设计哲学。

## 注意事项

- 协议适配器逻辑应与其他 Aha 业务**完全分离**，不得直接引用包括但不限于[统一配置系统](./模块开发/统一配置系统.md)、[订阅事件](./模块开发/订阅与发布事件.md)、[数据库](./模块开发/内置轮子与最佳实践/长效数据.md)、[计划任务](./模块开发/内置轮子与最佳实践/计划任务.md)等特性。若有需要参见 [Aha 服务调用](#aha-服务调用.md)。
- 发布的事件对象和 API 返回值应可 Pickle。

## 应具备的逻辑

- [注册为新的协议适配器并读取配置](#创建适配器)
- [与协议服务建立连接并监听协议事件](#建立连接并接收数据)
- [解析协议事件，依据协议所述语义挑选事件属性用以重新构成 Aha 事件对象并发布](#发布事件)
  - 对于消息事件，将消息内容解析为[消息序列](./数据结构/消息序列与消息段.md)结构。
- [将 API 调用转化为协议所需格式以调用聊天平台接口并返回值](#声明和调用-api)
  - 若参数涉及消息内容，将[消息序列](./数据结构/消息序列与消息段.md)结构转化为协议所需格式。
  - 涉及文件时处理文件传输。
  - 将返回值封装为 [Pydantic Model](https://docs.pydantic.dev/latest/concepts/models/)。
  - 若所需逻辑简单，在调用部分 API 之前先判断请求是否合法。

## 创建适配器

```python
# bots/example/__init__.py
from ..base import BaseBot

class Example(BaseBot):  # 继承 BaseBot 即注册为新的协议适配器
    platform = "QQ"  # 聊天平台名，对接相同平台的协议适配器之间应约定相同且唯一的平台名
    transport_class = ...

    @classmethod
    def _get_transport_kwargs(self, cfg: dict): ...
    async def _listen_callback(self, data): ...
```

## 建立连接并接收数据

Aha 搭建了[传输抽象层](./传输抽象层.md)。适配器需遵循传输层、协议转换层分离的逻辑结构。通过适配器的 `transport_class` 类属性指定传输层的类。

定义 `_get_transport_kwargs` 静态方法以从配置文件中的配置提取传输层所需参数。

接收到上报时会将数据传递给 `_listen_callback` 方法，数据类型因传输层而异。

### 示例

```python
import orjson

from core.transports import WebSocketClient

class Example(BaseBot):
    ...
    transport_class = WebSocketClient

    @classmethod
    def _get_transport_kwargs(cls, config: dict):
        if d := config.get("retry_config"):
            config["retry_config"] = cls.parse_retry_config(d)
        return config

    async def _listen_callback(self, data: bytes):
        self.logger.info(orjson.loads(data))
```

## 声明和调用 API

声明实例方法即声明 API，将 API 所需参数映射为方法参数；适配器接收到调用请求时会自动调用该方法并取其返回值。

应优先参考 Aha [已有 API](./模块开发/向协议服务请求/README.md) 的签名。

每个 API 方法接受的第一个位置参数为可用于标识请求并匹配响应的随机字符串，其次为 API 所需参数。

### bots.apis.BaseAPI

#### gen_id()

返回一个长度为 7 的随机十六进制字符串。

#### async add_schedule(args: AddScheduleArgs)

[创建长效计划任务](./模块开发/内置轮子与最佳实践/计划任务.md#schedadd_schedule--schedadd_persist_schedule)。

#### async rm_schedule_by_meta(meta: dict)

[删除长效计划任务](./模块开发/内置轮子与最佳实践/计划任务.md#schedrm_schedules_by_meta--schedrm_persist_schedules_by_meta)。

### bots.apis.Base*API

均为 `BaseAPI` 的子类，定义了一些聊天平台大概率具备的 API，还提供了部分 API 的默认实现。

> 定义 API 前可以先看看 `bots.apis` 中是否有现成的逻辑。比如 BaseMessageAPI.send_msg 就存在分流至 send_group_msg 和 send_private_msg 的逻辑。

### 示例

```python
from models.exc import APIException, APITimeoutError
from ...apis import BaseAccountAPI, BaseGroupAPI, BaseMessageAPI, BasePrivateAPI, BaseSupportAPI

class MessageAPI(BaseMessageAPI):  # 若欲定义的 API 过多，可以采用继承设计模式
    async def send_msg(self: Example, call_id, **kwargs):
        return await self._call_api(call_id, "send_msg", kwargs)

class Example(MessageAPI, BaseBot):
    ...

    def __init__(self, *args, **kwargs):
        super().__init__(*args, **kwargs)
        self._call_handlers: dict[str, Future] = {}

    async def _listen_callback(self, data: bytes):
        data = loads(data)
        if echo := data.get("echo"):  # 视协议层通过 ws 发送 API 返回值。
            self._call_handlers[echo].set_result(data)

    async def _call_api(self, echo, action, params=None, timeout=300) -> Any:
        future = self._call_handlers[echo] = get_running_loop().create_future()
        await self.transport.invoke(orjson.dumps({"action": action, "params": params or {}, "echo": echo}))
        try:
            data: dict = await wait_for(future, timeout)
            if (retcode := data["retcode"]) == 0:
                return data.get("data")
            raise APIException(data["message"], retcode)
        except TimeoutError:
            raise APITimeoutError()
        finally:
            del self._call_handlers[echo]
```

## Aha 服务调用

目前仅支持创建和依据 `metadata` 删除计划任务。

```python
from models.core import AddScheduleArgs, APSTriggerType

class Example(BaseBot):
    ...

    async def need_sched_api(self, call_id, **kwargs):
        schedule_args = AddScheduleArgs(  # 五个参数均必填，也均支持位置参数。
            api_method="send_msg",
            api_kwargs=kwargs,
            trigger=APSTriggerType.TIME_TRIGGER,
            trigger_kwargs={"seconds": 114514},
            schedule_kwargs={"metadata": {"platform": "QQ", "tag": "ban"} | kwargs},  # 注意 metadata 需避免默契冲突
        )
        await self.add_schedule(schedule_args)
```
